<objective>
Populate the curated settings registry with all hidden GNOME settings (those NOT exposed by GNOME Settings), and wire the detection engine into the window to dynamically build the sidebar from discovered categories with a settings count.

Purpose: This transforms Shadow Settings from hardcoded panels into a data-driven app. The registry IS the product's editorial voice -- it curates which settings are worth surfacing. The scanner validates each against the running system. The window rebuilds its sidebar from what's actually available.
Output: 6 registry files with ~65 curated settings, rewritten window.vala with dynamic sidebar population.
</objective>
