Create the Animator helper class and integrate cinematic animations:

**Animator** (`src/core/animator.vala`):
- Namespace: `ShadowSettings`
- Public class `Animator : Object`
- Static method `cascade_rows(Adw.PreferencesPage page)`:
  - Walk the page widget tree: page -> children -> PreferencesGroup -> children -> PreferencesRow instances
  - For each row: set `opacity = 0`, compute delay = `row_index * 40` ms
  - Use `GLib.Timeout.add(delay, ...)` to trigger an `Adw.TimedAnimation`:
    - `new Adw.TimedAnimation(widget, 0.0, 1.0, 250, callback_target)`
    - Easing: `Adw.Easing.EASE_OUT_CUBIC`
    - CallbackAnimationTarget sets `widget.opacity = value`
  - Return without animation (skip all) if the window has CSS class `reduce-motion`
  - Check reduce-motion: `widget.get_root().has_css_class("reduce-motion")` — if true, just ensure opacity=1 on all rows and return immediately
- Static method `pulse_modified(Gtk.Widget row)`:
  - Add CSS class `setting-glow` to the row (triggers CSS @keyframes amber-glow-pulse)
  - After 800ms timeout, remove `setting-glow` class (the `setting-modified` class provides the persistent indicator)
  - Skip if reduce-motion active
- Static method `spring_reveal(Gtk.Widget button)`:
  - Use `Adw.SpringAnimation` with `Adw.SpringParams(0.7, 1.0, 300.0)` (slight overshoot)
  - Animate opacity 0 -> 1 on the button
  - Skip if reduce-motion active

**CSS additions to `data/style.css`** (append to existing):
- `@keyframes amber-glow-pulse`: inset box-shadow from 0 opacity -> 0.8 -> settles at 0.35 (amber color). Duration handled by CSS `animation-duration: 800ms`.
- `.setting-glow` class: `animation-name: amber-glow-pulse; animation-duration: 800ms; animation-timing-function: ease-out; animation-fill-mode: forwards;`
- `window.reduce-motion *` and `window.reduce-motion row.setting-modified` and `window.reduce-motion .setting-glow`: `animation-duration: 0ms !important; animation-delay: 0ms !important; transition-duration: 0ms !important;`

**window.vala changes:**
- In the `sidebar_list.row_selected` handler, AFTER setting `content_stack.visible_child_name`, call `Animator.cascade_rows()` on the newly visible page (cast `content_stack.visible_child` to `Adw.PreferencesPage`). Skip for the "preferences" panel (it doesn't need cascade).
- NOTE: The cascade should run every time a panel is switched to, not just on first build. This makes panel navigation feel cinematic every time.

**meson.build:**
- Add `src/core/animator.vala` to the sources list

**Integration with WidgetFactory (DO NOT modify widget-factory.vala in this task):**
- The `setting-modified` CSS class is already applied by WidgetFactory. The CSS in style.css (from Plan 01) should already have a persistent indicator style for it (inset amber left border).
- The `setting-glow` class adds the one-shot pulse ON TOP of the persistent indicator. WidgetFactory's existing notify handlers are the place to call `Animator.pulse_modified()` but that requires modifying widget-factory.vala — defer this to a quick follow-up if needed. For now, the animation infrastructure is ready and the CSS classes are defined.
- Similarly, `Animator.spring_reveal()` is ready to be called when reset buttons appear. The WidgetFactory already creates reset buttons — calling spring_reveal on them can be wired later.
