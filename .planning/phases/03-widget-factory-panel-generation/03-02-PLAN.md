**Changes to the construct block:**

1. **Store categories and scanner as instance fields** so they're available for lazy construction:
   - Add private field: `private CategoryInfo[] categories;`
   - Add private field: `private SchemaScanner scanner;`
   - Add private field: `private HashTable<string, bool> panels_built;` (tracks which panels have been built)
   - Initialize `panels_built = new HashTable<string, bool>(str_hash, str_equal);`

2. **Keep existing pipeline** -- the scanner.scan() -> mapper.map() pipeline stays exactly the same.

3. **Remove eager page building** -- Delete the `foreach (var cat in categories)` loop that calls `build_category_page()` and `content_stack.add_named()`. Instead, only build the FIRST category page eagerly (Pitfall 6 from research -- avoids blank content on launch):
   ```vala
   // Build first category eagerly to avoid blank content area
   if (categories.length > 0) {
       var first_page = build_category_page_with_widgets(categories[0]);
       content_stack.add_named(first_page, categories[0].id);
       panels_built[categories[0].id] = true;
   }
   ```

4. **Update sidebar row_selected handler** for lazy construction:
   ```vala
   sidebar_list.row_selected.connect((row) => {
       if (row != null) {
           var action_row = (Adw.ActionRow) row;
           var panel_id = action_row.get_data<string>("panel-id");

           // Lazy build: construct page on first visit
           if (!panels_built.contains(panel_id) && panel_id != "power") {
               // Find the CategoryInfo for this panel_id
               foreach (var cat in categories) {
                   if (cat.id == panel_id) {
                       var page = build_category_page_with_widgets(cat);
                       content_stack.add_named(page, panel_id);
                       panels_built[panel_id] = true;
                       break;
                   }
               }
           }

           content_stack.visible_child_name = panel_id;
           split_view.show_content = true;
       }
   });
   ```

5. **Power/Flatpak gating stays identical** -- the existing `is_flatpak` check and PowerPanel instantiation remain exactly as they are (FR-7).

6. **Replace build_category_page method** -- Rename to `build_category_page_with_widgets` and rewrite to use WidgetFactory:
   ```vala
   private Adw.PreferencesPage build_category_page_with_widgets(CategoryInfo cat) {
       var page = new Adw.PreferencesPage();
       page.title = cat.title;
       page.icon_name = cat.icon;

       // Collect unique groups in order of appearance (same logic as before)
       string[] groups_seen = {};
       foreach (var def in cat.settings) {
           bool found = false;
           foreach (var g in groups_seen) {
               if (g == def.group) { found = true; break; }
           }
           if (!found) groups_seen += def.group;
       }

       // Build a PreferencesGroup per unique group
       foreach (var group_name in groups_seen) {
           var group = new Adw.PreferencesGroup();
           group.title = group_name;

           foreach (var def in cat.settings) {
               if (def.group == group_name) {
                   var widget = WidgetFactory.create_row(def, scanner);
                   if (widget != null) {
                       group.add(widget);
                   }
               }
           }

           page.add(group);
       }

       return page;
   }
   ```

7. **Delete the old build_category_page method entirely** -- it's fully replaced.

**What stays the same:**
- All sidebar building code (ActionRow creation, Power row, settings count subtitle)

- The scanner/mapper pipeline invocation
- Flatpak detection and Power panel handling
- First row auto-selection: `sidebar_list.select_row(sidebar_list.get_row_at_index(0))`
